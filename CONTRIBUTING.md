# Contributing to Heal-Io

Thank you for your interest in contributing to Heal-Io! This document provides guidelines for contributing to this educational project.

## Code of Conduct

- Be respectful and inclusive
- Focus on educational value
- Prioritize code quality and documentation
- Remember this is for academic/educational purposes

## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in [Issues](https://github.com/Harshnautiyal21/Heal-io/issues)
2. Create a detailed bug report including:
   - Description of the bug
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Screenshots if applicable
   - Environment details (OS, Docker version, etc.)

### Suggesting Enhancements

1. Check existing issues and pull requests
2. Create an enhancement issue with:
   - Clear description of the enhancement
   - Rationale and use case
   - Potential implementation approach
   - Any relevant examples or mockups

### Pull Request Process

1. **Fork the Repository**
   ```bash
   git clone https://github.com/Harshnautiyal21/Heal-io.git
   cd Heal-io
   git checkout -b feature/your-feature-name
   ```

2. **Make Your Changes**
   - Follow existing code style
   - Add comments for complex logic
   - Update documentation as needed
   - Test your changes thoroughly

3. **Commit Guidelines**
   ```
   feat: Add new feature
   fix: Fix bug
   docs: Update documentation
   style: Format code
   refactor: Refactor code
   test: Add tests
   chore: Update dependencies
   ```

4. **Push and Create PR**
   ```bash
   git push origin feature/your-feature-name
   ```
   - Create a Pull Request with a clear description
   - Reference any related issues
   - Wait for review and address feedback

## Development Guidelines

### Code Style

**Frontend (React/JavaScript)**
- Use functional components with hooks
- Follow ES6+ conventions
- Use meaningful variable and function names
- Add JSDoc comments for functions
- Use Tailwind CSS classes for styling

**Backend (Laravel/PHP)**
- Follow PSR-12 coding standards
- Use type hints
- Add PHPDoc comments
- Follow Laravel best practices
- Use dependency injection

**AI Service (Django/Python)**
- Follow PEP 8 style guide
- Use type hints (Python 3.10+)
- Add docstrings for functions and classes
- Keep functions focused and small

### Testing

- Add unit tests for new features
- Ensure existing tests pass
- Aim for >80% code coverage where applicable
- Test edge cases and error handling

### Documentation

- Update README.md if adding major features
- Add inline comments for complex logic
- Update API documentation for new endpoints
- Include examples in docstrings
